# 天织 ontop-aether 使用说明书

本文档基于当前仓库代码整理，说明 `ontop-aether` 的运行方式、页面能力、双后端架构、认证机制和部署形态。

## 1. 系统概述

`ontop-aether` 是一个基于 Ontop 的语义数据平台，用于把关系数据库映射成可查询、可发布、可供 AI 消费的虚拟知识图谱工作台。

核心能力：

- 数据源管理与 Bootstrap（自动生成本体 + 映射）
- 数据库结构概览
- SPARQL 查询与 SQL 重写
- 映射文件编辑与验证
- AI 自然语言问答
- 本体可视化
- 语义标注审核 + 合并到本体
- 业务词汇表（业务词 → 本体 URI 映射）
- 本体精化建议
- 数据发布（API / MCP / Tools）
- 平台治理（RBAC / API Key / 审计日志）
- AI 设置与系统设置
- 账号登录与会话管理

## 2. 双后端架构

系统采用 Java + Python 双后端架构，前端 `server.ts` 根据路径自动路由：

```
浏览器 → ontop-ui (Next.js :5000)
                │
                ├── /api/v1/datasources (CRUD)  → ontop-engine (Java :8081)
                ├── /api/v1/datasources/*/bootstrap → ontop-backend (Python :8000)
                ├── /api/v1/endpoint-registry    → ontop-engine (Java)
                ├── /api/v1/mappings             → ontop-engine (Java)
                ├── /api/v1/ontology             → ontop-engine (Java)
                ├── /api/v1/sparql/*             → ontop-engine (Java)
                ├── /api/v1/auth/*               → ontop-backend (Python)
                ├── /api/v1/ai/*                 → ontop-backend (Python)
                ├── /api/v1/annotations/*        → ontop-backend (Python)
                ├── /api/v1/glossary/*           → ontop-backend (Python)
                ├── /api/v1/suggestions/*        → ontop-backend (Python)
                ├── /api/v1/governance/*         → ontop-backend (Python)
                └── /api/v1/publishing/*         → ontop-backend (Python)
```

### 2.1 职责划分

| 服务 | 职责 | 技术栈 |
|------|------|--------|
| ontop-engine (Java) | 数据源 CRUD、端点切换、映射读写、TTL 解析、SPARQL 代理、查询历史 | Spring Boot + JdbcTemplate + OWLAPI |
| ontop-backend (Python) | Bootstrap 编排、AI/LLM 查询、语义标注、词汇表、精化建议、治理、认证、发布 | FastAPI + OpenAI SDK |
| ontop-endpoint | SPARQL 查询执行、SQL reformulate | Ontop 5.5.0 |

### 2.2 SQLite 共享

Java 和 Python 共享同一 SQLite 文件（WAL 模式）：

- Java 管理：datasources, endpoint_registry, query_history
- Python 管理：ai_config, semantic_annotations, business_glossary, users, sessions, ontology_suggestions, publishing_config, 治理表

数据源密码使用 Fernet 加密（Java 纯 JDK 实现，与 Python 零迁移兼容）。

## 3. 项目结构

```
ontop-aether/
├── ontop-ui/              # Next.js 前端
├── ontop-backend/         # FastAPI 后端（AI/LLM/治理）

```

## 4. 部署与启动

### 4.1 Docker Compose

在 `ontop-aether/` 目录下执行。

#### LVFA / Mondial 演示环境

```bash
docker compose -f docker-compose.lvfa.yml up -d --build
```

访问地址：

| 服务 | 端口 |
|------|------|
| 前端 | http://localhost:3001 |
| 后端 API 文档 | http://localhost:8001/docs |
| Java 引擎 | http://localhost:8083 |
| SPARQL Endpoint | http://localhost:18081 |
| PostgreSQL | localhost:5436 |

#### 默认 retail 环境

```bash
docker compose up -d --build
```

#### MySQL 演示环境

```bash
docker compose -f docker-compose.mysql.yml up -d --build
```

### 4.2 本地开发

#### Java 引擎

```bash
cd ontop-engine
mvn spring-boot:run
```

需要设置环境变量：`SQLITE_DB_PATH`, `ENCRYPTION_KEY_PATH`, `ONTOP_ENDPOINT_URL`。

#### Python 后端

```bash
cd ontop-backend
pip install -r requirements.txt
python3 -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

#### 前端

```bash
cd ontop-ui
pnpm install
pnpm dev
```

环境变量：`BACKEND_URL=http://localhost:8000`, `ENGINE_URL=http://localhost:8081`。

## 5. 认证与登录

### 5.1 当前认证机制

- 登录：`POST /api/v1/auth/login` → 返回 Bearer token
- 校验：`GET /api/v1/auth/me`
- 退出：`POST /api/v1/auth/logout`
- 改密：`POST /api/v1/auth/change-password`

前端 `AuthProvider` 维护会话，未登录自动跳转 `/login`。

### 5.2 双轨认证

平台支持密码登录和 API Key 认证（`oak_` 格式）。API Key 在「平台治理 → 访问控制」中管理。

### 5.3 默认账号

- 用户名：`admin`
- 密码：`admin123`

## 6. 页面说明

### 6.1 数据源管理 `/datasource`
- CRUD 操作（已迁至 Java）
- 测试连接（进程内调用，无 HTTP 开销）
- 数据库结构探查
- Bootstrap 发起（路由到 Python 后端，需 LLM）

### 6.2 SPARQL 查询 `/sparql`
- SPARQL 查询编辑器（代理到 Java → Ontop Endpoint）
- SQL reformulate 查看
- 查询历史管理

### 6.3 映射编辑 `/mapping`
- .obda 文件浏览、读取、编辑、保存
- 映射验证 + 端点重启

### 6.4 AI 助手 `/ai-assistant`
- 自然语言 → SPARQL，流式返回结果

### 6.5 本体可视化 `/ontology`
- .ttl 文件解析（OWLAPI），展示类/属性结构

### 6.6 语义标注 `/annotations`
- 默认跟随当前激活数据源
- LLM 标注审核 + 人工编辑 + 合并到本体

### 6.7 业务词汇表 `/glossary`
- 业务词 → 本体 URI 映射
- LLM 自动推导 + 人工维护

### 6.8 本体精化 `/refinement`
- LLM 分析本体结构，生成改进建议
- 支持一键应用低风险建议

### 6.9 数据发布 `/publishing`
- API Key / CORS / MCP 服务 / 工具定义生成

### 6.10 平台治理 `/governance`
- 项目管理：多租户、多项目、固定环境
- 访问控制：RBAC 角色/权限管理
- 审计日志：HTTP 请求自动记录

### 6.11 AI 设置 `/settings`
- LLM Provider 配置、模型选择、提示词编辑

### 6.12 系统设置 `/system`
- 用户信息、服务状态、Ontop 端点状态

## 7. 数据源上下文

部分页面涉及两层数据源概念：

- **激活数据源**：`endpoint_registry` 中 `is_current=1` 的记录，决定 Ontop 端点当前指向
- **页面选中数据源**：下拉切换查看其他数据源的数据（标注、词汇、精化、DB Schema）

## 9. 当前限制

- 认证为基础账号体系，非企业级 IAM / SSO
- 部分后端逻辑仍依赖 active endpoint（如标注合并）
- 平台治理处于基础阶段，尚未与外部 IAM 集成
- 文档截图可能滞后于 UI，以代码和运行页面为准
