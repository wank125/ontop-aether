# 开源本体数据映射工具梳理

> 对标 Microsoft Fabric IQ 的"数据→实体→关系"映射构建能力

---

## 一、本体建模（定义实体和关系）

| 工具 | 特点 | 链接 |
|------|------|------|
| **Protégé** | 最成熟的开源本体编辑器，支持 OWL/RDF，图形界面 | https://protege.stanford.edu/ |
| **OWLGrEd** | UML 风格的可视化 OWL 编辑器 | https://github.com/ozekik/awesome-ontology |

---

## 二、数据映射到本体（Data Binding 的核心）

对应 Fabric IQ 中"Entity Type → 表，Property → 字段"的绑定。基于 W3C 标准化方案：

### 映射语言标准

| 标准语言 | 适用范围 | 说明 |
|----------|----------|------|
| **R2RML** | 关系数据库 → RDF | W3C 标准（https://www.w3.org/TR/r2rml/） |
| **RML** | 异构数据源（CSV/JSON/XML/DB）→ RDF | R2RML 的泛化版本（https://rml.io/specs/rml/） |
| **YARRRML** | RML 的 YAML 简写形式 | 更易编写 |

### 开源执行引擎

| 工具 | 语言 | 支持标准 | 推荐场景 |
|------|------|----------|----------|
| **Morph-KGC** | Python | R2RML, RML, RML-star | **首选**，可扩展性好，支持大数据量 |
| **RMLMapper** | Java | RML | 学术研究，功能最全 |
| **RocketRML** | Node.js | RML | 高性能场景 |

### 工作流程

```
1. 用 Protégé 定义本体（OWL）
2. 用 YARRRML/R2RML 编写映射规则（表→实体，字段→属性）
3. 用 Morph-KGC 执行映射，生成 RDF 知识图谱
4. 导入三元组存储（GraphDB / Apache Jena / Neo4j）
```

---

## 三、图存储与查询（替代 Fabric IQ 的 Graph 引擎）

| 工具 | 类型 | 特点 |
|------|------|------|
| **Neo4j**（社区版） | 属性图数据库 | 最成熟，Cypher 查询语言，生态丰富 |
| **Apache Jena** | RDF 三元组存储 | W3C 标准语义，支持 SPARQL 推理 |
| **Memgraph** | 属性图数据库 | 开源，兼容 Neo4j Cypher，性能好 |
| **GraphDB**（免费版） | RDF 三元组存储 | 内置 OWL 推理引擎 |

---

## 四、Neo4j 生态中的本体集成方案

| 工具 | 功能 |
|------|------|
| **Neosemantics (n10s)** | Neo4j 插件，导入/导出 RDF/OWL 本体，支持语义映射 |
| **neo4j2owl** | OWL 2 本体 → Neo4j 图结构的语义保持映射 |
| **Neo4j Data Importer** | 内置可视化工具，CSV → 节点/关系的映射导入 |

---

## 五、与 Fabric IQ 的对应关系

| Fabric IQ 功能 | 开源替代方案 |
|----------------|-------------|
| 本体编辑器 | Protégé |
| Data Binding（数据绑定） | R2RML/RML + Morph-KGC |
| Graph 引擎 | Neo4j / Apache Jena |
| OWL 规则推理 | GraphDB（内置）/ Apache Jena |
| Data Agent（自然语言） | LangChain + Neo4j / GraphRAG |

---

## 六、推荐组合

**最小可行方案**：Protégé（建模）+ Morph-KGC（映射）+ Neo4j（存储查询）

这个组合可以完整复现 Fabric IQ 的核心链路：定义本体 → 绑定数据源 → 生成知识图谱 → 图查询。
